## Welcome to SuperMemory API

The SuperMemory API provides comprehensive programmatic access to your knowledge base. Whether you're building custom integrations, automating content workflows, or developing applications that leverage your stored knowledge, our RESTful API offers the tools you need.

## Base URL

All API requests should be made to:

```
https://api.vevibe.com
```

## Key Concepts

### Resources

The SuperMemory API is organized around several key resources:

- **Users**: Account information and settings
- **Spaces**: Knowledge organization containers
- **Memories**: Individual pieces of saved content
- **Chat**: AI-powered conversations with your knowledge
- **Search**: Semantic search across your content

### Authentication

All API requests require authentication using Bearer tokens:

```http
Authorization: Bearer your_api_key_here
```

### Content Types

SuperMemory supports various content types:

- **Pages**: Web pages and articles
- **Notes**: Text-based personal notes
- **Documents**: PDF and document files
- **Tweets**: Twitter content with special handling

## API Conventions

### HTTP Methods

| Method | Usage |
|--------|-------|
| `GET` | Retrieve resources |
| `POST` | Create new resources |
| `PATCH` | Update existing resources |
| `DELETE` | Remove resources |

### Response Format

All successful responses follow this structure:

```json
{
  "data": {
    // Response data here
  },
  "meta": {
    "timestamp": "2025-01-01T12:00:00Z",
    "requestId": "req_123456"
  }
}
```

### Error Format

Error responses include detailed information:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request parameters",
    "details": {
      "field": "content",
      "reason": "Content cannot be empty"
    }
  }
}
```

## Rate Limiting

The API implements rate limiting to ensure fair usage:

- **Free Plan**: 60 requests/minute, 10,000 requests/day
- **Pro Plan**: 300 requests/minute, 100,000 requests/day
- **Enterprise**: Custom limits based on agreement

Rate limit information is included in response headers:

```http
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 59
X-RateLimit-Reset: 1640995200
```

When rate limits are exceeded, you'll receive a `429 Too Many Requests` response with a `Retry-After` header.

## Common Use Cases

### Content Management

```javascript
// Add a new memory
const memory = await supermemory.memories.create({
  type: 'page',
  content: 'https://example.com/article',
  spaces: ['work-space-id']
});

// Search memories
const results = await supermemory.search({
  query: 'machine learning best practices',
  limit: 10
});
```

### AI Integration

```javascript
// Chat with your knowledge
const response = await supermemory.chat({
  messages: [
    { role: 'user', content: 'What have I learned about React hooks?' }
  ]
});

// Stream the response
for await (const chunk of response) {
  console.log(chunk.content);
}
```

### Space Management

```javascript
// Create a new space
const space = await supermemory.spaces.create({
  name: 'Research Project',
  isPublic: false
});

// Invite collaborators
await supermemory.spaces.invite(space.uuid, {
  email: 'colleague@example.com',
  accessType: 'edit'
});
```

## API Endpoints Overview

### User Management
- `GET /v1/user` - Get user information
- `GET /v1/user/spaces` - Get user's spaces
- `POST /v1/user/key` - Generate API key

### Space Management
- `GET /v1/spaces` - List spaces
- `POST /v1/spaces/create` - Create space
- `GET /v1/spaces/{id}` - Get space details
- `PATCH /v1/spaces/{id}` - Update space
- `DELETE /v1/spaces/{id}` - Delete space

### Memory Management
- `GET /v1/memories` - List memories
- `GET /v1/memories/{id}` - Get memory details
- `DELETE /v1/memories/{id}` - Delete memory
- `POST /v1/memories/batch-delete` - Bulk delete

### Content Operations
- `POST /v1/add` - Add new content
- `POST /v1/bulk-add` - Bulk add content
- `POST /v1/reprocess-missing-metadata` - Reprocess content

### AI & Search
- `POST /v1/search` - Semantic search
- `POST /v1/chat` - AI chat
- `GET /v1/recommended-questions` - Get suggestions
