## Migration Guides

### Upgrading to v2.0

<Warning>
SuperMemory 2.0 includes breaking changes. Please review the migration guide before upgrading.
</Warning>

#### API Changes
- **Base URL**: Updated from `supermemory.ai/api` to `api.vevibe.com`
- **Authentication**: New API key format (regeneration required)
- **Endpoints**: Several endpoint paths have changed

#### Migration Steps
1. **Backup Data**: Export your memories before upgrading
2. **Update API Keys**: Generate new API keys for integrations
3. **Update Integrations**: Point to new API endpoints
4. **Test Functionality**: Verify all features work correctly

#### Compatibility
- **Browser Extensions**: Auto-update to compatible versions
- **Mobile Apps**: Update required from app stores
- **Third-party Tools**: Check with tool maintainers for compatibility

### Database Migration

Existing users will be automatically migrated:

```sql
-- Automatic migration script (example)
ALTER TABLE memories ADD COLUMN content_hash VARCHAR(64);
ALTER TABLE spaces ADD COLUMN collaboration_settings JSONB;
CREATE INDEX idx_memories_content_hash ON memories(content_hash);
```
